Getting Started

The whole workflow starts with two steps:

  1. Load Photos — choose a folder. Every supported image inside it (including subfolders) is added to your library.
  2. Generate — the app automatically distributes your photos across as many pages as needed, in a randomised mosaic-style layout.

Before you press Generate

A handful of settings in the sidebar shape what Generate actually produces — worth setting these deliberately rather than leaving them at their defaults:

  • Number of Pages — think of this as a maximum, not a fixed target. Generate stops either once it reaches this many pages, or once it runs out of photos, whichever happens first. If you set this too low for a large photo library, some photos will be left unused rather than automatically spilling onto extra pages — if you want every photo included, make sure Number of Pages is comfortably high enough for your library size (or raise Photos per Page instead, since that also affects how many photos each page can absorb).
  • Photos per Page (Min/Max) — each page gets a random number of photos within this range, which is what gives the mosaic its varied, organic feel rather than a rigid uniform grid. A tighter range (e.g. 8–10) gives more consistent-looking pages; a wider range (e.g. 4–16) gives more visual variety page to page.
  • Paper/Page Size — set this before generating, since it directly affects how photos get cropped to fill each frame. Changing page size after the fact still works, but frames and photos will need to be re-fitted to the new proportions. See Page Setup below for the full list of sizes, including custom presets.
  • Orientation and Photo Order — orientation controls whether pages are portrait, landscape, or a mix; photo order controls whether photos are placed in filename order or shuffled randomly across pages.

From there, every page is fully editable — nothing about the automatic layout is locked in. Think of Generate as a fast, high-quality starting point, not a final answer.

Tip: if you don't love a particular layout, use Reshuffle to regenerate that page with a fresh random arrangement, without touching your other pages.
Margins tip, especially if printing via a photobook service: many commercial photobook printers trim or fold the outer edge of each page during binding, which can quietly crop a few millimetres off — most noticeably at the binding edge. If you're printing through a service like this rather than on your own printer, it's worth setting a larger-than-default margin, particularly on the side that will sit in the binding, so nothing important (faces, text, the edge of a photo) ends up lost in the trim.

Crucially, how much gets trimmed varies from one manufacturer to the next — there's no single set of margins that's right everywhere, so treat any specific values as a starting point and run your own small test print before committing to a large order.

As a reference point, these sidebar margin settings have been tested and work well with the following services:

  • Photobox — Top 20, Bottom 20, Left 80, Right 54.
  • Colorland — Top 30, Bottom 30, Left and Right the same as Photobox (Left 80, Right 54).
  • Printerpix — works with either of the above.

Load vs Add, and Photos vs Files

There are four buttons for getting images into your library, and they differ along two separate lines:

  • Load replaces your whole library — it clears everything currently loaded and starts fresh with your new selection. Add appends to what's already there, without removing anything (duplicates, matched by filename, are skipped automatically).
  • The Photos buttons (📂 Load Photos… / ➕ Add Photos…) pick a whole folder — every supported image inside it, including subfolders, comes in at once. The Files buttons (🖼 Load Files… / 🖼+ Add Files…) let you pick individual image files instead, for when you only want a specific handful rather than an entire folder.

So, put simply: use Load Photos to start a project from a folder, Add Photos to fold in another folder later, and the two Files variants when you'd rather hand-pick individual images than point at a folder.

Note for Windows users: the folder picker on Windows only lets you choose folders, not individual files (and won't list the files inside a folder while you browse — that's normal, not a fault). The Load Files… and Add Files… buttons are the way to select individual photos there. On Mac either style of button can pick files or a folder, so the Files buttons are just a convenience.

Editing a Page

Click any frame to select it. A small toolbar appears above it with icons for:

  • 📷 Pick photo — choose a specific photo for this frame from your library
  • 🔄 Rotate — rotates the photo 90° at a time (only shown once a frame has a photo)
  • 🔀 Swap — swap this frame's photo with another frame's
  • ✕ Delete — removes the frame entirely

Moving and resizing frames

  • Resize — drag the small handle at a frame's corner
  • Move a frame's position — hold Alt and drag the frame
  • Pan/zoom the photo inside its frame — scroll to zoom, drag to reposition (without Alt)
  • Add a new frame — use the Add Frame button, then position and size it like any other
Tip: frames snap to nearby edges and centres as you drag, for clean alignment — the snap distance is adjustable in settings if you want it looser or tighter.

Text Boxes

Add a text box from the toolbar, then click it to edit. Each text box has two parts:

  • Title (optional) — sits at the top like a heading, with its own size, bold, and italic controls, independent of the body
  • Body — the main text underneath, which is genuinely vertically centred within whatever space is left below the title

Style controls cover font, size, colour, alignment, line height, letter spacing, background, and drop shadow — all found in the side panel once a text box is selected.

Borders

Frame Border (in settings) has two independent toggles — one for photos, one for text boxes — sharing a single thickness slider. Turn on just the one you want; they don't have to match.

Page Setup

Page size options include standard sizes (A3, A4, A5, US Letter), square formats (8×8", 10×10"), and a fully custom size in millimetres. If you use a custom size often, save it as a named preset so it's available in the dropdown from then on.

Margins, gap between frames, and snap distance are all adjustable, and can be saved as your personal defaults (⚙ Defaults) so every new project starts exactly how you like it. If you're printing through a photobook service rather than at home, see the margin/binding tip in Getting Started before settling on your margin values.

Saving Your Work

There are three distinct ways to save, and they solve different problems:

OptionWhat it savesBest for
SaveThe layout only, as a small JSON file — photos are referenced by filename, not includedQuick saves during a session on the same computer
ArchiveThe layout and a full copy of every photo used, bundled into one ZIPMoving a project between computers, or long-term safekeeping
Save as TemplateThe layout only, with every photo reference deliberately removedReusing a favourite design with a completely different set of photos

Why Archive is often the better choice

A plain Save is small and fast, but it only remembers your photos by name — it depends on those exact files still being in the same folder, with the same names, whenever you come back to restore it. If you reorganise your photo library, rename files, or move to a different computer, a plain Save can leave you hunting for the right folder.

Archive avoids that problem entirely, because your photos travel inside the file itself. Restoring an archive doesn't require finding anything — everything needed is already there in the ZIP. That makes it the more genuinely reliable option, especially for a project you care about keeping long-term, or one you might want to reopen on a different computer.

The trade-off: because Archive physically includes every photo, the file size scales with your library. A project built from a few dozen photos archives to a modest file — but a project using hundreds of full-resolution photos can produce a genuinely large ZIP file, taking longer to create and more disk space to store. For very large libraries, a plain Save (paired with keeping your original photo folder somewhere safe) may be the more practical everyday choice, with an occasional Archive as a more durable checkpoint.

A reasonable approach: use Save for your regular day-to-day working file, and make an Archive once a project is finished, or before any major reorganisation of your photo folders — that way you always have one fully self-contained, future-proof copy, without paying the file-size cost on every single save.

Restoring a Project

Use Restore to reopen a saved session. If any referenced photos aren't currently loaded, the app asks you to select the folder containing them — and if your project drew photos from more than one folder, it keeps asking, letting you load folder after folder until everything's found (or you choose to continue with gaps).

Missing photos

If some photos genuinely can't be found, the sidebar shows their exact filenames under a "Missing" heading — rather than just a bare count — so you have something concrete to search for.

If you later load or add photos that match a missing name, they're automatically placed back into their original frame, at the same crop, zoom, and rotation they had when you saved — no manual re-placing needed.

Tip: Archive sidesteps all of this, since there's never anything to go looking for — see the Saving section above.

Export & Print

Export Page and Export All save your pages as JPEG images (a ZIP of JPEGs when exporting multiple pages).

Print (labelled Save for Printing in the app) prepares your page(s) as JPEG and asks where to save them — open the saved file(s) in your photo viewer (Preview on Mac, Photos on Windows) and print from there with Cmd+P (Mac) or Ctrl+P (Windows). This two-step approach is necessary because neither platform's native app framework allows an app like this one to open a live print dialog directly, but it gives you full control over paper size and scaling through your photo viewer's own print options.

Tips for Large Photo Libraries

The app has been tested with libraries in the range of 1,000+ photos across 400+ pages, so don't hesitate to work at real scale. A few things worth knowing at that size:

  • Generating and exporting hundreds of pages takes real time — the progress bar reflects genuine per-page work, not a placeholder.
  • Photos are automatically scaled to properly fill their frame even on pages you never manually open — you don't need to click through every page before exporting.
  • If exporting produces a different page count than expected, the app will tell you directly rather than silently shipping an incomplete result.

Roughly how long should this take?

Saving, Archiving, and Exporting all write to disk at a similar rate — measured at roughly 10–11 MB/s during testing. That figure is IPC round-trip overhead, not raw disk or CPU speed, so it's a reasonable planning number across different machines.

For Export Page, Export All, and Print, a single rendered page at 300 DPI is far smaller than a full Archive — roughly 8–16MB for A4 (2480×3508px) and 18–30MB for A3 (3508×4961px), depending on how busy the layout is. At the write speed above, that's only 1–3 seconds of actual disk time per page. For a large multi-page export, most of the total time is spent rendering each page (drawing every photo into its frame and encoding the JPEG) rather than writing it to disk — rendering speed scales with photos-per-page and your computer's processor, not disk throughput, so it isn't covered by the figures above.

Note: these figures come from real measurements on one machine and will vary with your own disk, processor, and how visually complex each page is — treat them as a rough planning guide, not a guarantee.

Troubleshooting

Red "⚠ DPI" badge on a photo

This means that photo is being stretched beyond what looks sharp at print size in that particular frame — the badge shows the approximate effective resolution. Zoom out slightly, use a higher-resolution photo, or use a smaller frame or page size.

HEIC photos not loading

HEIC/HEIF (the default iPhone photo format) requires native decoding support. macOS provides this, so HEIC/HEIF photos work normally on Mac. Windows does not, so on Windows these photos are skipped (and, when restoring an Archive, the app will tell you how many were left blank for this reason). If you're seeing HEIC photos skipped, convert them to JPEG first — most photo apps offer this as an export option.

A frame's photo looks off-centre with a border on

This was a known issue in earlier versions, now fixed — text and photo positioning both account for border thickness. If you're seeing this, make sure you're on the latest version.
